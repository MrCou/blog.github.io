高可用服务注册中心

发表于 2018-03-19 | 分类于 spring-cloud

之前服务注册中心 说明了服务注册server、client以及一些列的功能，但是这个时候服务中心是单一的，如果出问题，就会造成很严重的后果，服务中心也可以互相注册，形成服务中心的高可用。

准备

首先准备两个服务注册中心，

# 第一个注册中心
server:
  port: 1112
eureka:
  instance:
    ip-address: 127.0.0.1
#    hostname: server2
  client:
    service-url:
    # 通过ip访问
      defaultZone: http://127.0.0.1:1111/eureka
     # 通过hostname访问 
#      defaultZone: http://server1:1111/eureka


# 第二个注册中心
server:
  port: 1111
eureka:
  instance:
    ip-address: 127.0.0.1
#    hostname: server2
  client:
    service-url:
      defaultZone: http://127.0.0.1:1112/eureka
#      defaultZone: http://server1:1112/eureka

客户端

多个服务中心都用上

eureka.client.serviceUrl.defaultZone=http://127.0.0.1:1111/eureka,http://127.0.0.1:1112/eureka

服务中心之间最好是相互注册，形成闭环
